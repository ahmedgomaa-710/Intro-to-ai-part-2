LECTURE 04

Problem Formulation & Search Strategies

What is Search?

We consider the problem of designing goal-based agents in environments that are: fully observable, deterministic, discrete, and known.

The agent must find a sequence of actions that reaches the goal. The performance measure is defined by:

  • Reaching the goal
  • How "expensive" the path to the goal is
Search = the process of looking for a sequence of actions that leads from the initial state to a goal state.

ما هو البحث؟

نصمم وكلاء قائمة على الهدف في بيئات: مرئية بالكامل، حتمية، منفصلة، ومعروفة.

الوكيل لازم يلاقي تسلسل من الأفعال يوصّله للهدف. معيار الأداء بيتحدد بـ:

  • الوصول للهدف
  • تكلفة المسار — مدى "غلاء" الوصول للهدف
البحث = عملية البحث عن تسلسل أفعال يوصّل من الحالة الابتدائية للحالة الهدف.

Problem Formulation — 5 Components

A problem is defined by five items:

1. Initial State

The starting state of the agent.

2. Actions

Set of possible actions available to the agent.

3. Transition Model

Describes what each action does — also called the Successor function.

4. Goal Test

Determines if a given state is the goal state.

5. Path Cost

Assigns a numeric cost to each path. Usually 1 per step.

The State Space is the set of all states reachable from the initial state by any sequence of actions — represented as a directed graph.

A Solution is a sequence of actions leading from the initial state to a goal state.

صياغة المشكلة — ٥ مكونات

المشكلة بتُعرَّف بـ خمسة عناصر:

١. الحالة الابتدائية

نقطة البداية للوكيل.

٢. الأفعال

مجموعة الأفعال المتاحة للوكيل.

٣. نموذج الانتقال

بيصف نتيجة كل فعل — تُسمى أيضاً دالة الخلف.

٤. اختبار الهدف

بيحدد إذا كانت الحالة الحالية هي الهدف.

٥. تكلفة المسار

بتعيّن تكلفة رقمية لكل مسار. عادةً ١ لكل خطوة.

فضاء الحالات هو مجموعة الحالات القابلة للوصول من الحالة الابتدائية — يُمثَّل كرسم بياني موجّه.

الحل هو تسلسل من الأفعال من الحالة الابتدائية للحالة الهدف.

Problem Formulation Examples

Vacuum Cleaner

States: agent location + square status
Initial: any random state
Actions: Left, Right, Suck
Goal test: all squares clean
Path cost: 1 per step

N-Puzzle (8-Puzzle)

States: locations of tiles
Initial: any random arrangement
Actions: move blank L/R/U/D
Goal test: ordered arrangement
Path cost: 1 per move

Tower of Hanoi

States: disk locations on 3 pegs
Initial: all disks on peg 0
Actions: move disk between pegs
Goal test: all disks on peg 2
Path cost: 1 per move

Search Tree & Strategy

State vs. Node:

  • State = a physical configuration of the world
  • Node = a data structure in the search tree containing: state, parent node, action, path cost g(n), depth

A search strategy is defined by the order of node expansion.

Strategies are evaluated by:

Completeness

Does it always find a solution if one exists?

Time Complexity

How many nodes are generated/expanded?

Space Complexity

How many nodes are kept in memory?

Optimality

Does it always find the least-cost solution?

Key parameters: b = branching factor, d = depth of shallowest solution, m = maximum depth of state space

شجرة البحث والاستراتيجية

الحالة vs. العقدة:

  • الحالة (State) = تكوين جسدي للعالم
  • العقدة (Node) = هيكل بيانات في شجرة البحث يحتوي: الحالة، العقدة الأم، الفعل، تكلفة المسار g(n)، العمق

استراتيجية البحث بتُحدَّد بترتيب توسيع العقد.

التعقيد الزمني

التعقيد المكاني

كام عقدة بتتخزَّن في الذاكرة؟

الأمثلية

هل دايماً بيلاقي الحل الأقل تكلفة؟

المعاملات الأساسية: b = معامل التفرع، d = عمق أضحل حل، m = الحد الأقصى لعمق فضاء الحالات

LECTURE 05–06

Uninformed Search — BFS & UCS

Blind vs. Heuristic Strategies

Blind (Uninformed) Search

Has no additional information about states beyond the problem definition. Can only generate successors and distinguish goal from non-goal states.

Heuristic (Informed) Search

Exploits extra information to assess that one node is "more promising" than another — smarter but needs domain knowledge.

Uninformed search algorithms: BFS, UCS, DFS, DLS, IDS, Bidirectional Search

بحث أعمى مقابل بحث إرشادي

البحث الأعمى (Uninformed)

ما عندوش معلومات إضافية عن الحالات. بس يقدر يولِّد خلفاء ويعرف حالة الهدف.

البحث الإرشادي (Heuristic)

بيستغل معلومات إضافية يقيّم بيها العقدة الأكثر وعداً — أذكى بس محتاج معرفة بالمجال.

خوارزميات البحث الأعمى: BFS, UCS, DFS, DLS, IDS, Bidirectional

Breadth-First Search (BFS)

Strategy: Always expand the shallowest unexpanded node.

Data structure: FIFO Queue — new successors go to the end.

How it works: Explores all nodes at depth d before moving to depth d+1.

Expansion order (example A→B,C→D,E,F,G...): A, B, C, D, E, F, G

PropertyBFS ResultNotes
Complete?YesIf b is finite
Time?O(bd)Exponential in d
Space?O(bd)Keeps every node in memory!
Optimal?YesOnly if all step costs = 1
Space is BFS's main problem — more critical than time. With b=10 and d=12: ~111 terabytes of memory needed!

البحث بالعرض أولاً (BFS)

الاستراتيجية: دايماً يوسِّع أضحل عقدة غير موسَّعة.

هيكل البيانات: Queue من نوع FIFO — الخلفاء الجدد بيروحوا للآخر.

الطريقة: بيستكشف كل العقد في العمق d قبل ما يروح لـ d+1.

الخاصيةنتيجة BFSملاحظة
اكتمال؟نعملو b محدودة
الزمن؟O(bd)أسي في d
المساحة؟O(bd)بيحتفظ بكل العقد في الذاكرة!
أمثل؟نعمبس لو تكلفة كل خطوة = ١
المساحة هي المشكلة الرئيسية لـ BFS — أخطر من الزمن. مع b=10 و d=12: ~١١١ تيرابايت من الذاكرة!

Uniform-Cost Search (UCS)

Strategy: Always expand the least-cost unexpanded node.

Data structure: Priority Queue — ordered by path cost g(n).

Key difference from BFS: BFS uses step count; UCS uses actual path cost. Better when step costs are not equal.

Let C* = cost of optimal solution, ε = minimum step cost.

PropertyUCS ResultNotes
Complete?YesIf g > ε (step costs > 0)
Time?O(bC*/ε)Processes all nodes with cost ≤ C*
Space?O(bC*/ε)Same as time
Optimal?YesAlways finds cheapest path
UCS is equivalent to BFS when all step costs are equal. UCS is optimal but can be slow when C*/ε is large.

البحث بالتكلفة الموحدة (UCS)

الاستراتيجية: دايماً يوسِّع أقل تكلفة عقدة غير موسَّعة.

هيكل البيانات: Priority Queue — مرتَّب حسب تكلفة المسار g(n).

الفرق عن BFS: BFS بيستخدم عدد الخطوات؛ UCS بيستخدم التكلفة الفعلية. أفضل لما تكاليف الخطوات مش متساوية.

C* = تكلفة الحل الأمثل، ε = الحد الأدنى لتكلفة الخطوة.

الخاصيةنتيجة UCSملاحظة
اكتمال؟نعملو g > ε (تكاليف الخطوات موجبة)
الزمن؟O(bC*/ε)بيعالج كل العقد بتكلفة ≤ C*
المساحة؟O(bC*/ε)نفس الزمن
أمثل؟نعمدايماً يلاقي أرخص مسار
UCS يساوي BFS لو كل تكاليف الخطوات متساوية. UCS أمثل بس ممكن يبطأ لو C*/ε كبير.
ملخص المحاضرة ٥–٦ — BFS & UCS 📖 البحث بالعرض أولاً والبحث بالتكلفة الموحدة

🌊 ٢. BFS — البحث بالعرض أولاً

➤ الفكرة: دايماً وسِّع أضحل عقدة غير موسَّعة

➤ هيكل البيانات: FIFO Queue (الجديد يروح للآخر)

➤ الترتيب: طبقة طبقة — كل العمق d قبل d+1

➤ Fringe/Queue: بيبدأ بـ [A] ← يوسِّع ← يضيف [B,C] ← يوسِّع B ← يضيف [C,D,E] إلخ

➤ Complete: نعم (لو b محدودة)

➤ Time: O(b^d) — أسي في عمق الحل

➤ Space: O(b^d) — المشكلة الرئيسية! بيحتفظ بكل العقد في الذاكرة

➤ Optimal: نعم — بس بشرط تكلفة كل خطوة = ١

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💰 ٣. UCS — البحث بالتكلفة الموحدة

➤ الفكرة: دايماً وسِّع أقل تكلفة عقدة غير موسَّعة

➤ هيكل البيانات: Priority Queue (مرتَّب بالتكلفة g(n))

➤ الفرق عن BFS: BFS يعد الخطوات، UCS يحسب التكلفة الفعلية

➤ متى يُستخدم؟: لما تكاليف الخطوات مش متساوية

➤ C* = تكلفة الحل الأمثل، ε = الحد الأدنى لأي تكلفة خطوة

➤ Complete: نعم (لو g > ε)

➤ Time: O(b^(C*/ε)) — بيعالج كل العقد بتكلفة أقل من C*

➤ Space: O(b^(C*/ε)) — نفس الزمن

➤ Optimal: نعم — دايماً يلاقي أرخص مسار

📊 ٤. مقارنة BFS و UCS

➤ UCS = BFS لو كل تكاليف الخطوات متساوية

➤ BFS أسرع لما الخطوات متساوية، UCS أدق لما مش متساوية

➤ كلهم أمثليين وكاملين — لكن الذاكرة مشكلة كبيرة!

📱 ٥. تطبيقات BFS في الحياة

➤ GPS Navigation: Google Maps بيستخدم BFS للمسار من نقطة لأخرى

➤ شبكات الحاسوب: تطبيقات P2P (زي التورنت) تلاقي ملف باستخدام BFS

➤ الشبكات الاجتماعية: Facebook يعامل كل مستخدم كعقدة ويبحث بـ BFS

➤ التكنولوجيا اللاسلكية: MIMO بيستخدم BFS لإيجاد أقصر مسافة

LECTURE 07–08

Uninformed Search — DFS, DLS, IDS & Bidirectional

Depth-First Search (DFS)

Strategy: Always expand the deepest unexpanded node.

Data structure: LIFO Stack — put successors at the front.

How it works: Goes deep into one branch until it can't go further, then backtracks.

PropertyDFS ResultNotes
Complete?NoFails in infinite-depth spaces or loops
Time?O(bm)May explore entire tree
Space?O(b·m)Only stores path + siblings
Optimal?NoMay find a non-optimal solution
DFS's big advantage: Space! O(b·m) vs BFS's O(b^d). Much more memory-efficient.

البحث بالعمق أولاً (DFS)

الاستراتيجية: دايماً يوسِّع أعمق عقدة غير موسَّعة.

هيكل البيانات: LIFO Stack — الخلفاء بيروحوا للأول.

الطريقة: بيروح في عمق فرع لحد ما ما يقدرش يكمل، بعدين بيرجع (Backtrack).

الخاصيةنتيجة DFSملاحظة
اكتمال؟لابيفشل في الفضاءات اللانهائية والحلقات
الزمن؟O(bm)ممكن يستكشف الشجرة كلها
المساحة؟O(b·m)بس يحتفظ بالمسار + الإخوة
أمثل؟لاممكن يلاقي حل غير أمثل
الميزة الكبرى لـ DFS: المساحة! O(b·m) مقارنة بـ O(b^d) لـ BFS. أكثر كفاءة في الذاكرة بكتير.

Depth-Limited Search (DLS)

Idea: DFS with a depth limit L — nodes at depth L have no successors generated.

Solves: The infinite-depth problem of DFS.

Problem: If d > L, the goal won't be found!

Let d = goal depth, L = depth limit value.

PropertyDLS ResultNotes
Complete?NoFails if d > L
Time?O(bL)Depends on limit L
Space?O(b·L)Linear in L
Optimal?NoMay miss cheaper solutions

البحث بعمق محدود (DLS)

الفكرة: DFS مع حد أقصى للعمق L — العقد عند العمق L ما بيتولَّدلهاش خلفاء.

يحل: مشكلة العمق اللانهائي في DFS.

المشكلة: لو d > L، الهدف مش هيتلاقى!

d = عمق الهدف، L = قيمة حد العمق.

الخاصيةنتيجة DLSملاحظة
اكتمال؟لابيفشل لو d > L
الزمن؟O(bL)بيعتمد على الحد L
المساحة؟O(b·L)خطي في L
أمثل؟لاممكن يفوّت حلول أرخص

Iterative Deepening Search (IDS / IDFS)

Idea: Repeatedly apply DLS with increasing limit: L=0, 1, 2, 3, ... until the goal is found.

Best of both worlds: Combines BFS completeness + BFS optimality with DFS space efficiency.

Downside: Re-expands nodes from previous iterations — but the overhead is small!

PropertyIDS ResultNotes
Complete?YesLike BFS
Time?O(bd)Same as BFS asymptotically
Space?O(b·d)Like DFS — much better than BFS!
Optimal?YesIf all step costs equal
IDS is the preferred uninformed search method when search space is large and the depth of solution is unknown!

البحث بالتعمق التكراري (IDS)

الفكرة: بيطبِّق DLS بتكرار مع حد متزايد: L=0, 1, 2, 3, ... لحد ما يلاقي الهدف.

أفضل ما في الاتنين: اكتمال BFS + أمثلية BFS + كفاءة مساحة DFS.

العيب: بيعيد توسيع العقد من التكرارات السابقة — بس التكلفة الإضافية صغيرة!

الخاصيةنتيجة IDSملاحظة
اكتمال؟نعمزي BFS
الزمن؟O(bd)نفس BFS بشكل مقارب
المساحة؟O(b·d)زي DFS — أفضل بكتير من BFS!
أمثل؟نعملو تكاليف الخطوات متساوية
IDS هو الخيار المُفضَّل في البحث الأعمى لما فضاء البحث كبير وعمق الحل مجهول!

Bidirectional Search (BS)

Idea: Run two simultaneous BFS searches — one forward from START, one backward from GOAL.

Solution: Found when the two frontiers meet in the middle.

Why faster? Instead of b^d, we do 2 × b^(d/2) — massive savings!

PropertyBS ResultNotes
Complete?YesIf b is finite
Time?O(bd/2)Huge improvement over BFS!
Space?O(bd/2)Both frontiers stored
Optimal?YesIf all step costs equal
Example: d=12, b=10 → BFS needs 10^12 = 1 trillion nodes. Bidirectional needs only 2×10^6 = 2 million!

البحث ثنائي الاتجاه (BS)

الفكرة: بيشغِّل بحثين BFS في نفس الوقت — واحد للأمام من START وواحد للخلف من GOAL.

الحل: بيتلاقى لما الحدّان يلتقيوا في المنتصف.

ليه أسرع؟ بدل b^d، بنعمل 2 × b^(d/2) — توفير هائل!

الخاصيةنتيجة BSملاحظة
اكتمال؟نعملو b محدودة
الزمن؟O(bd/2)تحسين هائل عن BFS!
المساحة؟O(bd/2)الحدّان بيتحفظوا
أمثل؟نعملو تكاليف الخطوات متساوية
مثال: d=12, b=10 → BFS محتاج 10^12 = تريليون عقدة. الثنائي محتاج بس 2×10^6 = مليونين!

⬇️ ١. DFS — البحث بالعمق أولاً

➤ الفكرة: دايماً وسِّع أعمق عقدة غير موسَّعة

➤ هيكل البيانات: LIFO Stack (الخلفاء بيروحوا للأول)

➤ Stack الابتدائي: [A] ← يوسِّع A ← [B,C] ← يوسِّع B ← [D,E,C] ← يوسِّع D ← [H,I,E,C] إلخ

➤ Complete: لا — بيفشل في الفضاءات اللانهائية والحلقات

➤ Time: O(b^m) — ممكن يستكشف الشجرة كلها

➤ Space: O(b·m) — ميزة كبيرة! بس يحتفظ بالمسار الحالي + إخوته

➤ Optimal: لا — ممكن يلاقي حل غير أمثل

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🚧 ٢. DLS — البحث بعمق محدود

➤ الفكرة: DFS مع حد L — العقد عند العمق L ما بيتولَّدلهاش خلفاء

➤ يحل مشكلة: العمق اللانهائي في DFS

➤ المشكلة: لو d > L الهدف مش هيتلاقى!

➤ Complete: لا (إذا d > L)

➤ Time: O(b^L)

➤ Space: O(b·L)

➤ Optimal: لا

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔄 ٣. IDS — البحث بالتعمق التكراري (الأفضل في البحث الأعمى!)

➤ الفكرة: يكرر DLS مع حد متزايد: L=0, 1, 2, 3, ... لحد ما يلاقي الهدف

➤ الميزة: يجمع اكتمال BFS + أمثلية BFS + كفاءة مساحة DFS

➤ العيب: يعيد توسيع العقد — بس التكلفة الإضافية صغيرة (حوالي b/(b-1) ضعف)

➤ Complete: نعم

➤ Time: O(b^d) — نفس BFS بشكل مقارب

➤ Space: O(b·d) — أفضل بكتير من BFS!

➤ Optimal: نعم (لو تكاليف الخطوات متساوية)

➤ IDS هو الخيار المُفضَّل لما فضاء البحث كبير وعمق الحل مجهول

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

↔️ ٤. Bidirectional Search — البحث ثنائي الاتجاه

➤ الفكرة: بيشغِّل BFS من البداية وBFS من الهدف في نفس الوقت

➤ الحل: لما الحدّان يلتقيوا في المنتصف

➤ ليه أسرع؟ بدل b^d نعمل 2 × b^(d/2) — فرق هائل!

➤ Complete: نعم

➤ Time: O(b^(d/2))

➤ Space: O(b^(d/2))

➤ Optimal: نعم (لو تكاليف الخطوات متساوية)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
